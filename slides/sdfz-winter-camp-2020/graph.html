邻接表

常用的存图方式：

  • 链表：每个节点的邻接表用一个边链表表示。每条边内部都有一个 next 表示链表中的下一条边。
    • 可以用一个边数组 e[] 来存所有的边，用一个 head[] 数组存链表的第一条边。
    • 反向边可以存在相邻的位置，这样 e[i] 和 e[i^1] 就是一对反向边。常用于网络流。
    • 缺点：非连续访问内存。稠密图上可能有性能问题。
  • vector 存储：同一个节点相邻的所有的边在内存中是连续的。
    • 理论上在开 O2/O3 的环境下，访问 vector 内的元素会被编译器内联展开为直接访问数组。图的遍历的效率会比邻接链表高。
    • push_back 由于涉及到内存的动态分配，所以比较比较慢，无论开不开优化，尽管复杂度是均摊 O(1)\mathrm O(1)O(1) 的。可以考虑使用 reserve 提前准备好空间。
    • 总结：不开优化的情况慎用。

邻接矩阵

用 G[u][v] 来表示 uuu 和 vvv 之间是否有边相连。通常可以记录边的长度，或者是重边的数量。一般是给稠密图用的。

对于无向图应该有 G[u][v] = G[v][u]。

既然是个矩阵，考虑下矩阵乘法：G⋅G=G2G \cdot G = G^2G⋅G=G2：

Gu, v2=∑z∈VGu, z⋅Gz, vG^2_{u,\ v} = \sum_{z \in V} G_{u,\ z} \cdot G_{z,\ v} Gu, v2​=z∈V∑​Gu, z​⋅Gz, v​

实际上是一个 DP 转移方程：枚举中间点 zzz，uuu 到 zzz 的路径条数乘上 zzz 到 vvv 的路径条数的值贡献给 uuu 到 vvv 的路径条数。因此 Gu, v2G^2_{u,\ v}Gu, v2​ 是图中 uuu 到 vvv 长度为 222 的路径数量。不难归纳出 Gu, vkG^k_{u,\ v}Gu, vk​ 就是图中 uuu 到 vvv 长度为 kkk 的路径条数。

矩阵乘法可以快速幂计算。

度数

度数 d(u)d(u)d(u) 即与顶点 uuu 相关联的边的条数。如果是自环，一般会认为对顶点度数有 222 的贡献。对于有向图，则分别有出度和入度的概念。

定理　图的度数总和一定是偶数。

因为一条边对两个端点各有一次贡献。

定理　如果一张无向图中仅有两个点 uuu、vvv 的度数为奇数，则 uuu 和 vvv 之间必有一条路径相连。

如果没有，则 uuu 和 vvv 分居在两个连通块内，则这两个连通块的各自的度数总和为奇数，矛盾。

定理　设简单无向图 GGG 中的最小度数为 δ(G)δ(G)δ(G)，则 GGG 中一定有一个包含 δ(G)+1δ(G) + 1δ(G)+1 个点的简单回路。

首先，由于 GGG 是简单图，所以 GGG 至少有 δ(G)+1δ(G) + 1δ(G)+1 个点。

之后，从 GGG 中任意一个点出发做 DFS，每次选择与当前点相邻的 δ(G)δ(G)δ(G) 个点中未访问过的点走过去，这样至少能走 δ(G)δ(G)δ(G) 步，于是得到了一个长度为 δ(G)+1δ(G) + 1δ(G)+1 的简单环。

欧拉图

存在一个边不重复的回路，经过图中每条边一次。点可以经过任意多次。这样的回路称为欧拉（Euler）回路。

如果存在欧拉回路，则环上每个点必定是进入几次就会出来几次，因此图中每个点的度数必须为偶数。

定理　对于无向连通图 GGG，GGG 中存在欧拉回路当且仅当 GGG 中每个点的度数为偶数。

**充分性：**注意图中每个点的度数都是偶数，所以 GGG 一定不是一棵树，因此 GGG 中必定存在一个环 CCC。

将环 CCC 中的所有边删去，剩下的图 G′G'G′ 中每个点的度数依然为偶数。考虑对图的边数进行归纳法，则可假设 G′G'G′ 存在一个欧拉回路 C′C'C′，于是可以考虑将 CCC 从某一点剪断后插入到 C′C'C′ 中，于是就得到了 GGG 的一个欧拉回路。

同时由这个证明可以得出，一个欧拉图可以拆分成若干个环的并。

---3

如果是有向图，则对应的，图中每个点的入度应该等于出度。同样这个条件也是充分的。

定理　对于无向单连通图 GGG，GGG 中存在欧拉回路当且仅当 GGG 中每个点的入度等于出度。

证明方法和无向图是类似的。

欧拉回路中删去一条边就变为欧拉路径。不难推出图中存在欧拉路径的充要条件。

+++

如果要求出欧拉回路，首先要检查图是否满足上述定理的要求。之后，可以从图中任意一个点开始做 DFS，DFS 的过程中任意选择一条没有走过的边走下去。这样如果把所有边都走了一遍，那么我们就得到了一个欧拉回路。

不过这种无脑走法不一定就能把所有边都遍历到。例如：

如果从左上角的点出发，到达中间的点之后，可能会往左边走，最后回到原点。这样的话就会忽略了右边的三条边。

因此需要在 DFS 回溯的时候检查一下是否还有其它边没有走，如果有就需要继续走下去，并且需要将之前走出来的环和新出来的环合并一下。

写合并可以使用链表，但是比较麻烦。考虑到插入一个新环就是把之前的环的后半部分挪一下，然后在这中间把新环放进去，由于我们知道最终的欧拉回路里面一定有 mmm 条边，因此不如在回溯的时候再一条一条地把边倒序放在一个大小为 mmm 的数组内。

**当前弧优化：**由于在 DFS 的过程中会重复进入一个点多次，我们可以拿一个数组 cur[] 存下每个点当前已经考虑到哪条边了，避免邻接表中边的重复访问。

n⩽1000n \leqslant 1000n⩽1000。

Hamilton 图

与欧拉回路不同，Hamilton 回路要求是经过图中每个点有且仅有一次的回路。图中的边不一定需要经过。但是求解 Hamilton 回路十分困难，除非有特殊条件，否则无法在多项式时间内求解。一般图的 Hamilton 回路（包括更难的旅行商问题）可以用状压 DP 解决。

如果图中存在一个 Hamilton 回路，那么如果我们删掉图中 kkk 个点，Hamilton 回路就会分为至多 kkk 个部分，因此删点后的图中至多有 kkk 个连通块。这是存在 Hamilton 回路的一个必要条件。

定理　对于无向图 GGG，若 GGG 存在 Hamilton 回路，则对任意 S⊆VS \subseteq VS⊆V，G−SG - SG−S 的连通块数量不超过 ∣S∣|S|∣S∣。

左图中删去两个红色点后，图中有三个连通块，不满足定理的条件，因此左图没有 Hamilton 回路。

右图是著名的 Peterson 图。它虽然满足定理的条件，但是它也没有 Hamilton 回路。

介绍一个存在 Hamilton 回路的充分条件。

定理　GGG 为无向图，若 GGG 中存在两点 uuu 和 vvv，满足 uuu、vvv 间没有边相连，并且 d(u)+d(v)⩾nd(u) + d(v) \geqslant nd(u)+d(v)⩾n，则 GGG 有 Hamilton 回路等价于 G+uvG + uvG+uv 有 Hamilton 回路。

若 GGG 有 Hamilton 回路，显然加上一条边的 G+uvG + uvG+uv 也会有。

格雷码

考过今年 CSP 的应该都知道啥是格雷码吧。

考虑边长为 111 的 nnn 维立方体，每个顶点的坐标是一个长度为 nnn 的 0/10/10/1 序列，不难发现一条棱两端的顶点坐标只有一位不相同。因此 nnn 阶格雷码序列对应于 nnn 维立方体上的一个 Hamilton 回路。

竞赛图 Hamilton 回路

竞赛图是完全有向图，相当于将完全无向图 KnK_nKn​ 中每条边都指定了一个方向后得到的图。我们知道 KnK_nKn​ 中肯定有 Hamilton 回路和 Hamilton 路径。实际上对于竞赛图也有相应的结论。

定理　nnn 个点的竞赛图中一定有一条长度为 nnn 的简单路径，即 Hamilton 路径。

图论中很多构造性的证明都是使用数学归纳法，这个证明也不例外。我们首先假设 n−1n - 1n−1 个点的竞赛图中存在 Hamilton 路径，那么考虑 nnn 个点的竞赛图 GGG，从 GGG 中任意删去一个点 uuu，则得到一个 n−1n - 1n−1 个点的竞赛图 G′G'G′。由归纳假设， G′G'G′ 存在 Hamilton 路径 v1→v2→v3→⋯vn−1v_1 → v_2 → v_3 → \cdots v_{n - 1}v1​→v2​→v3​→⋯vn−1​。

在图 GGG 中，如果边 u→v1u → v_1u→v1​ 存在，则 u→v1→⋯→vn−1u → v_1 → \cdots → v_{n - 1}u→v1​→⋯→vn−1​ 就是 GGG 的一个 Hamilton 路径。否则就存在边 v1→uv_1 → uv1​→u。之后我们考虑边 u→v2u → v_2u→v2​，如果有，那么 v1→u→v2→⋯→vn−1v_1 → u → v_2 → \cdots → v_{n - 1}v1​→u→v2​→⋯→vn−1​ 是 Hamilton 路径，不然的话边 v2→uv_2 → uv2​→u 就存在。依次类推，如果上述过程一直找不到 uuu 的合适的出边，那么一定有边 vn−1→uv_{n - 1} → uvn−1​→u，于是将 uuu 放在之前的 Hamilton 路径的末尾即可。

需要注意，如果没有其它条件，竞赛图中不一定存在 Hamilton 回路。但是如果有 Hamilton 回路，那么说明竞赛图必定是强连通的。不出意外，这个也是充分条件。

---2

定理　nnn 个点的强连通的竞赛图中一定有 Hamilton 回路。

同样是使用归纳法。设竞赛图为 GGG，从 GGG 中随意删掉一个点 uuu 后得到 G′G'G′。考虑对 G′G'G′ 进行缩点，我们注意到缩点后的图必定是一条链 C1→C2→C3→⋯→CkC_1 → C_2 → C_3 → \cdots → C_kC1​→C2​→C3​→⋯→Ck​，其中 CiC_iCi​ 表示一个强连通分量。根据归纳假设，每个 CiC_iCi​ 内部都有一个 Hamilton 回路，因此我们只要思考如何将这些回路接起来即可。

由于 GGG 是强连通的，因此必定有从 CkC_kCk​ 到 uuu 的边以及从 uuu 到 C1C_1C1​ 的边。因此我们可以从 uuu 出发，依次走过 C1C_1C1​、C2C_2C2​、...、CkC_kCk​ 的 Hamilton 回路即可。注意可能需要调整离开 CkC_kCk​ 时的终点，确保最后能够走回 uuu。

Floyd 算法

当需要图中所有点对间的最短路时，大多都会涉及到这个算法。

算法直接使用邻接矩阵存图，G[u][v] 表示图中 uuu 到 vvv 的边的长度。如果 uuu 到 vvv 没有边，则设为 ∞\infin∞。一般代码里面会设做一个很大的值，诸如 10910^9109、101810^{18}1018。或者是 0x3f3f3f3f，因为这个可以直接用 memset(G, 0x3f, sizeof(G)) 来清空，并且其两倍（0x7e7e7e7e）没有超出 int 的范围（0x7fffffff）。

算法就是三个 for 循环：首先枚举中介点 kkk，之后更新所有点对 iii、jjj 之间的最短路。

Bellman-Ford / SPFA

包括之前的 Floyd 算法，几乎所有的最短路算法都是依靠松弛操作实现的。即对于任意三个点 xxx、yyy、zzz，如果有：

δ(x, y)+δ(y, z)<δ(x, z)δ(x,\ y) + δ(y,\ z) < δ(x,\ z) δ(x, y)+δ(y, z)<δ(x, z)

则可以使用 δ(x, y)+δ(y, z)δ(x,\ y) + δ(y,\ z)δ(x, y)+δ(y, z) 来更新 δ(x, z)δ(x,\ z)δ(x, z)。如果不能再更新了，则最短路已经求出。即最短路满足所谓的 “三角不等式”：

δ(x, y)+δ(y, z)⩾δ(x, z)δ(x,\ y) + δ(y,\ z) \geqslant δ(x,\ z) δ(x, y)+δ(y, z)⩾δ(x, z)

---1

Bellman-Ford 算法是单源最短路算法：给出源点 sss，求出所有的 δ(s, u)δ(s,\ u)δ(s, u)。算法就是进行 n−1n - 1n−1 次迭代，每次迭代检查图中的每条边，看能不能进行松弛操作。初始时设源点到源点的距离 δ(s, s)δ(s,\ s)δ(s, s) 为 000，到其它点 uuu 的距离 δ(s, u)δ(s,\ u)δ(s, u) 为 ∞\infin∞。因为最短路肯定都是简单路径，n−1n - 1n−1 次迭代后所有的最短路一定都能枚举得到，除非图中有负环。

如果想判断是否有负环，则可以再迭代一次，看是否能更新现在的最短路。如果能则说明出现了负环。

这样做的复杂度是 O(nm)\mathrm O(nm)O(nm) 的。但实际上里面有很多没必要的松弛操作。例如，如果一个点 uuu 的距离 δ(s, u)δ(s,\ u)δ(s, u) 在上次迭代中没有变动，那么下一次迭代中从 uuu 出发的边都没必要再考虑了。SPFA 是 Bellman-Ford 的队列优化，即把最短距离变动了的所有点都放在一个队列中，然后每次从队首取出一个点进行更新。初始时只用将源点放入队列。

SPFA 同样也可以判定负环：如果有一个点重复入队达到 nnn 次，则说明出现了负环。

众所周知，SPFA 尽管在随机图上跑得很快，但还是可以卡成 O(nm)\mathrm O(nm)O(nm) 的。建议如无把握，最短路的题目尽量不要使用 SPFA。

Dijkstra 算法

与 SPFA 有着相似的代码框架，但不同在于，Dijkstra 算法每次选取 δ(s, u)δ(s,\ u)δ(s, u) 最小的 uuu 来进行松弛。期间需要使用一个堆来维护每个点到源点的最短路，每次从堆中取出最近的点即可。可以想象成是源点是一处水源，边的长度是水流经过这条边所需要花费的时间，于是每个节点的最短路就是水流第一次到达某个节点的时刻。本质上是一个时间轴上的扫描线。

Dijkstra 算法的一个特点就是，当一个点从堆中 pop 出来的时候，其最短路的距离就已经固定了。因此每个点只用更新一次。

不过当图中有负权边的时候 Dijkstra 算法会有问题，就因为经过负权边后被更新的最短路不一定就比之前更新的最短路要长，就像是有后效性一样。

通常大家都是用 priority_queue 来写的。这个东西是 vector 上的二叉堆，一般问题不大，复杂度基本上是 O((n+m)log⁡m)\mathrm O((n + m) \log m)O((n+m)logm)。

众所周知使用 Fibonacci 堆来写复杂度就是 O(m+nlog⁡n)\mathrm O(m + n \log n)O(m+nlogn)，因为其支持 O(1)\mathrm O(1)O(1) 的插入和 O(1)\mathrm O(1)O(1) 的 decrease。但是其常数巨大，因此 OI 基本上派不上用场。
